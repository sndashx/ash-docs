ASH — Phase 01: Renderer Foundation
====================================

Phase goal: a truecolor terminal renderer that fills the viewport
with a colored grid, supports camera, sub-cell motion, lighting,
animation, and a moving cursor controlled by the keyboard.

Entry criteria: Phase 0 complete.
Exit criteria: ./build/ash --render-test shows a colored viewport
              with a smoothly moving cursor.

[ENTRY CRITERIA]
  - Phase 0 complete.

[EXIT CRITERIA]
  - ./build/ash --render-test fills terminal with dark blue.
  - Cursor cell moves smoothly with arrow keys.
  - Window resize (SIGWINCH) handled without crash.
  - Light grid visibly darkens an area when a torch is removed.
  - Sub-cell movement: hold right arrow 5s, cursor moves 80 cells
    with no jitter.

[RISKS]
  - Terminal size detection across platforms. Mitigation: ioctl on
    Unix, fallback to env vars.
  - Truecolor support varies. Mitigation: detect via $COLORTERM or
    OSC 4 probe; degrade to 256-color.
  - Render performance at 240x80. Mitigation: dirty-rect diffing.
  - Mouse escape sequence parsing complexity. Mitigation: feature
    flag, default off.
