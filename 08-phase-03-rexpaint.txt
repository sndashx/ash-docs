ASH — Phase 03: REXPaint Pipeline
===================================

Phase goal: load and write .xp files. World loader spawns entities
from layers per Appendix A.

Entry criteria: Phase 2 complete.
Exit criteria: a 5x5 test .xp file loads into a Map, entities
              spawn from layers, save roundtrip is byte-identical.

[ENTRY CRITERIA]
  - Phase 2 complete.

[EXIT CRITERIA]
  - Load 5x5 test .xp -> Map struct.
  - Spawn entities per layer convention.
  - Write Map struct -> .xp file byte-identical to input.
  - Roundtrip preserves all cell data.

[RISKS]
  - .xp format quirks (column-major, gzip header). Mitigation:
    test against known REXPaint output.
  - Sidecar JSON consistency. Mitigation: schema validation.

Step 0300.00.00. Create src/rexpaint/xp_format.hpp.
  - Sub: constexpr int XP_MAGIC = ... (REXPaint magic number).
  - Sub: constexpr int XP_VERSION = ... (REXPaint version).
  - Sub: struct XpHeader { int version; int width; int height;
         int layer_count; };

Step 0300.00.01. Document .xp binary format.
  - Sub: Magic 8 bytes (specific pattern).
  - Sub: Version 4 bytes int.
  - Sub: Width 4 bytes int.
  - Sub: Height 4 bytes int.
  - Sub: Layer count 4 bytes int (1-9).
  - Sub: For each layer, column-major:
    - For x in [0, width):
      - For y in [0, height):
        - glyph 4 bytes uint32 (UTF-32)
        - fg 3 bytes RGB
        - bg 3 bytes RGB

Step 0300.00.01. Create src/rexpaint/xp_loader.hpp.
  - Sub: XpHeader read_header(std::istream&).
  - Sub: std::vector<Cell> read_layer(std::istream&, XpHeader const&,
         int layer_idx).

Step 0300.00.01. Implement XpHeader read_header.
  - Sub: Read 8 bytes, verify magic.
  - Sub: Read 4 bytes version.
  - Sub: Read width, height, layer_count.
  - Sub: Validate layer_count in [1, 9].

Step 0300.00.02. Implement read_layer.
  - Sub: Allocate width*height Cell vector.
  - Sub: For x in [0, width):
    - For y in [0, height):
      - Read 4 bytes glyph.
      - Read 3 bytes fg.
      - Sub: 3 bytes bg.
