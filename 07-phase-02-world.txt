ASH — Phase 02: World Data Model
==================================

Phase goal: an entt-based world representation. Maps hold cells,
entities, components. Spatial queries work. Triggers fire.

Entry criteria: Phase 1 complete.
Exit criteria: a 5x5 test map with 3 entities (player, NPC, item)
              loads; player can walk into the NPC's trigger and
              see it fire in the log.

[ENTRY CRITERIA]
  - Phase 1 complete.

[EXIT CRITERIA]
  - Map struct loads from struct (not yet from .xp).
  - Entity components registered.
  - Player can move with collision detection.
  - NPC has schedule (stub) and disposition.
  - Item can be picked up and added to inventory.
  - Trigger fires on entry; logs event.

[RISKS]
  - entt signal handling and lifecycle. Mitigation: clear ownership
    of registry.
  - Collision detection cost at large entity counts. Mitigation:
    spatial hash grid (later optimization).
